Github Blog 만드는 중 발생한 에러 경험 (Hugo, PaperMod)

February 28, 2022

Github Hugo Blog 를 만들기 위해 설치하는 중 발생한 에러에 대해 기술하겠습니다.

좀 더 나은 해결방법, 좀 더 간결하고 깔끔한 해결방법이 있을 수 있겠지만 제 시행착오를 그대로 공유합니다.

누군가는 저와 같은 현상을 겪으실수도 있으니까요.

1. 블로그 설치 중 에러 발생 구간#

Github Blog 만들기 (Hugo, PaperMod)

제가 작성한 이전 글(위 링크로 이동가능)에서 5. github에 업로드하여 블로그 게시하기 > 4번 항목에서 에러가 났습니다.

2. 삽질 과정 공유#

Windows Terminal에서의 기록들을 작성하겠습니다.

PS D:\Project\blog> git init
Initialized empty Git repository in D:/Project/blog/.git/

PS D:\Project\blog> git remote add origin https://github.com/exodus0315/blog.git  # 현재 이 줄까지는 정상 작동

PS D:\Project\blog> git submodule add -b master https://github.com/exodus0315/exodus0315.github.io.git public  # 이 코드 입력부터 에러 발생
Cloning into 'D:/Project/blog/public'...
warning: You appear to have cloned an empty repository.
fatal: 'origin/master' is not a commit and a branch 'master' cannot be created from it
fatal: unable to checkout submodule 'public'
Fetching origin

보시면 아시겠지만, 4번째 줄 코드를 작성하고부터 주의, 에러 문구가 여러개 나왔습니다…

아마… master 브랜치가 없으며, public을 체크아웃하지 못한다는 것 같았습니다.

저는 깃허브 관련 지식이 전무하기 때문에 구글링하면서 아무거나 하나씩 따라해보았습니다.

PS D:\Project\blog> git checkout public  # checkout 할 수 없다고 하길래 체크아웃해보았습니다. 정답은 아닌 것 같습니다.
error: pathspec 'public' did not match any file(s) known to git

PS D:\Project\blog> git submodule add -b master https://github.com/exodus0315/exodus0315.github.io.git public
fatal: 'public' does not have a commit checked out  # public에 체크아웃된게 없다고 합니다.

PS D:\Project\blog> cd public  # public 폴더 진입

PS D:\Project\blog\public> git add .  # 체크아웃된게 없다고 하니까 해당 명령어 작성

PS D:\Project\blog\public> cd ..  # 이전 폴더로 빠져나오기

PS D:\Project\blog> git submodule add -b master https://github.com/exodus0315/exodus0315.github.io.git public
fatal: 'public' does not have a commit checked out  # 생각해보니 public 폴더가 비어있으니 git add할 것도 없었죠...

PS D:\Project\blog> hugo -t PaperMod  # public 폴더가 비어있어서 체크아웃 못하는 것 같아서 빌드 진행했습니다.
Start building sites …  # 빌드 과정 중 발생한 메시지는 생략하겠습니다.

PS D:\Project\blog> cd public  # 이제 public 폴더에 파일들이 생겼을테니 public 폴더 진입

PS D:\Project\blog\public> git add .  # public 폴더 안에 있는 모든 파일들을 추가
warning: LF will be replaced by CRLF in categories/index.html.  # git add 하니 이런 경고 메시지가 많이 등장했습니다.
The file will have its original line endings in your working directory

# CRLF라고 해서 플랫폼(OS) 간 차이 때문인데 아래 코드를 입력하여 해결할 수 있습니다. 자세한 설명은 생략합니다.
PS D:\Project\blog\public> git config --global core.autocrlf true

# 이제 아래 코드를 입력해도 에러 메시지가 발생 안 했습니다.
PS D:\Project\blog> git submodule add -b master https://github.com/exodus0315/exodus0315.github.io.git public

PS D:\Project\blog\public> git commit -m "first commit"  # 커밋 메시지 입력

PS D:\Project\blog\public> cd ..  # 이전 폴더 진입

# 혹시 몰라서 아래 코드를 한번 더 입력해봤습니다. 이제 public 폴더가 이미 존재한다고 뜹니다.
PS D:\Project\blog> git submodule add -b master https://github.com/exodus0315/exodus0315.github.io.git public
fatal: 'public' already exists in the index

PS D:\Project\blog> cd public  # public 폴더에서 git add 한 것들을 이제 push 하기 위해 폴더 진입

PS D:\Project\blog\public> git push origin master  #깃 푸쉬 진행
error: src refspec master does not match any  # 아마... master 브랜치가 없어서 생긴 현상 같았습니다.
error: failed to push some refs to 'https://github.com/exodus0315/exodus0315.github.io.git'

PS D:\Project\blog\public> git branch -M master  # 마스터 브랜치 생성

PS D:\Project\blog\public> git push origin master  # 푸시 성공

발생한 로그들은 많이 생략했습니다.

git 명령어도 써본 적이 없기 때문에 쏟아져나오는 에러, 경고, 로그 메시지에 엄청 당황했습니다.

github는 혼자 공부한다고 써봤지만 git 명령어가 아닌 GitHub Desktop을 써본 정도였습니다.

내가 어디를 잘못한지 몰라 폴더, 리포지토리를 전부 삭제 후 다시 시도해보기도 몇 번 해봤습니다.

똑같은 짓을 반복하고, 조~금씩 다르게도 시도해보면서 도전했더니 시간상으로 몇시간 정도 걸린 것 같습니다…

전체적으로 부족한 점이 많았지만 결과적으로는 브랜치에 대한 이해가 부족해서 발생한 현상인 것 같습니다…

아마 쌩초보자 분들은 저처럼 당황하실 수 있으니 제 삽질 경험을 공유해두겠습니다.

사실 블로그를 만든 이유가 이 삽질 때문입니다.

검색 실력이 부족해서인지, 관련 글들이 적어서인지 만족할만한 정확한 답변을 얻지 못했습니다.

아마 영어 실력이 부족해서일수도 있겠죠.

아무튼 저와 같은 분들에게 도움이 되었으면 합니다.
