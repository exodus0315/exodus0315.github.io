Github Blog 만들기 (Hugo, PaperMod)

February 27, 2022

이번에 블로그를 만들면서 겪었던 방법과 시행착오들을 기록하려고 합니다.

구글에 검색해보면 많은 깃허브 블로그 관련글들이 있지만, 제가 원하는 글은 적었습니다.

특히나 Mac OS에서의 설명이 많았고, 초보자 입장에서 글이 잘 이해가 안되어서 저도 글을 적어봅니다.

OS : Windows 10 64-bit

Used SW : Git, Windows Terminal, VSCode

SSG : Hugo

Theme : PaperMod

많은 정적 사이트 생성기(SSG, Static Site Generator)이 있지만 저는 Hugo를 활용하기로 마음먹었습니다.

많은 이유들이 있겠지만 빠르고, ruby보다는 go라는 언어에 대해 관심도 좀 더 있었고, Jamstack Site Generators에서 많은 별점도 받고 있었습니다.

결정적으로는… 개발에 대해 아직 아무것도 모르는 입장에서 “내가 원하는 테마가 있는가?” 도 중요한 것 같습니다.

저는 Hugo Themes 사이트에서 PaperMod 라는 테마가 마음에 들어 위 이유와 함께 최종적으로 휴고(Hugo)를 선택했습니다.

동적 사이트 vs 정적 사이트, jekyll, Hugo 등 정적 사이트 생성기의 장단점은 따로 서술하지 않겠습니다.

github blog에서 사용가능한 정적 사이트 생성기들이 많으니 비교해보시면 되겠습니다.

1. Git 설치#

Git Download

위 사이트에서 PC에 맞는 OS에 맞게 설치하시면 되겠습니다.

설치를 진행한 뒤 명령 프롬프트(cmd)에서 아래 메시지를 입력하셔서 enter 누르시면 버전을 보실 수 있습니다.

git version

만약 버전이 안 나오시면 설치가 잘못 된 것입니다.

참고로 저는 앞으로 cmd 대신 Windows Terminal 을 기준으로 설명드리겠습니다. (Windows Terminal 설치)

2. Hugo 설치#

Hugo Download

  1. 위 사이트에 들어간 뒤 스크롤을 약간 내리신 뒤 Assets 항목에서 OS에 맞는 파일을 다운로드받는다.

  2. C:\hugo\bin 폴더에 압축을 해제한다. (해당 폴더가 없다면 폴더를 만들어서 진행)

  3. 윈도우 검색(Widnows키 + Q)에서 “시스템 환경 변수 편집"을 검색한 뒤, 하단의 [환경 변수]를 클릭한다.

  4. 하단 시스템 변수 > Path 선택 > [편집] 버튼 클릭 > [새로 만들기] 버튼 클릭 > C:\hugo\bin 추가

  5. Windows Terminal에서 hugo version 을 쳐서 제대로 설치되었는지 확인하기

※ 위 내용은 Install Hugo > Windows에서 확인하실 수 있습니다. (영상 참고 가능)

대부분은 공식 사이트에서 해당 내용들을 확인할 수 있습니다. (영어라는 단점이 있지만요…)

3. Hugo 프로젝트 만들기#

아래 내용들은 hugo-PaperMod Installation을 참고하여 작성하였습니다.

테마 별로 설치 가이드 페이지가 대부분 있으니 따라하시면 됩니다.

  1. 원하는 폴더에 들어가서 마우스 우측 클릭 후 [Windows 터미널에서 열기]를 클릭한다.
  1. Windows Terminal에서 아래 코드를 입력한다. 저는 <프로젝트 이름>에 blog 라고 작성하였습니다.
hugo new site blog -f yml  # hugo new site <프로젝트 이름> -f yml
  1. blog 라는 폴더가 만들어졌으면 아래 코드를 입력하여 폴더에 진입한다.
cd blog  # cd <프로젝트 이름>
  1. hugo-PaperMod Installation 사이트에 들어가서 3가지 방법 중 하나를 선택하여 진행한다.

Method 1 : git clone 활용 (code를 복사-붙여넣기 하면 알아서 설치 진행)

Method 2 : git submodule 활용 (code를 복사-붙여넣기 하면 알아서 설치 진행)

Method 3 : 압축 파일을 다운로드 받아 직접 적용 (저는 3번 방법을 썼기 때문에 3번 방식대로 서술하겠습니다.)

  1. Method 3번 기준, 압축 파일을 blog/themes 폴더에 압축 해제한다.

저는 폴더 이름을 PaperMod 로 수정하였습니다. (최종 폴더명 : blog/themes/PaperMod)

  1. 테마 적용 후, blog 폴더에 있는 config.yml을 열어 Sample config.yml에 들어가서 code를 복사-붙여넣기한 뒤 저장한다.

4. 글 작성하기 & 로컬 서버 실행하기#

  1. Windows Terminal에서 아래 코드를 입력하여 새 포스트(새 글)를 생성해본다.
hugo new --kind post test.md  # hugo new --kind post <file name>
  1. Windows Terminal에서 아래 코드를 입력하여 서버를 실행해본다.
hugo server
  1. 인터넷 브라우저 주소창에 “localhost:1313” 을 입력하여 잘 실행되는지 확인한다.

5. github에 업로드하여 블로그 개시하기#

  1. github에 2개의 리포지토리(repository)를 생성한다.

1번째 리포지토리 : blog

2번쨰 리포지토리 : <깃허브 아이디>.github.io (제 기준, exodus0315.github.io)

  1. 혹시 blog 폴더에 public 폴더가 있다면 미리 삭제해준다.

  2. Windows Terminal에서 아래 코드를 입력한다. (blog 폴더 경로에서 진행되어야 함)

git init
git remote add origin <1번째 리포지토리 주소>

# 리포지토리 주소 규칙은 일반적으로 https://github.com/<유저이름(user name)>/<리포지토리 이름>.git
# 깃허브에서 리포지토리에 들어가면 주소를 쉽게 확인 및 복사-붙여넣기 가능
  1. Windows Terminal에서 아래 코드를 입력한다.
git submodule add -b master <2번째 리포지토리 주소> public
  1. Windows Terminal에서 아래 코드를 입력하여 테마를 빌드해준다.
hugo -t PaperMod  # hugo -t <테마 이름>
  1. 아래 코드를 순서대로 진행하여 깃허브에 빌드된 테마들을 업로드한다.
cd public  # public 폴더로 진입
git add .
git commit -m "<커밋 메세지>"
git push origin master

cd ..  # 이전 폴더로 진입 (blog 폴더로 돌아감)
git add .
git commit -m "<커밋 메세지>"
git push origin master
  1. 이후 인터넷 브라우저 주소창에 <유저이름>.github.io를 입력하여 블로그에 접속한 뒤 확인한다.

순서대로만 따라하면 깃허브 블로그를 만들 수 있도록 쉽게 작성하려고 했는데 글솜씨가 부족해서인지 어렵네요.

이후에는 설치 중 발생한 에러, 간단한 테마 설정에 대해 글을 쓰도록 하겠습니다.
